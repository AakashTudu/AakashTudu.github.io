The Operational Shift

Moving from "Staff Memory" to "System Logic".

Current Risks

  • 1. Inconsistent staff practices & manual workarounds.

The Solution

  • 2. Automated Logic: Rules embedded in code.

  • 4. Locked Audit Trails: Files secure after QA.

Beyond the Specification

Future-Proofing LEAD

My role isn't just to build what you asked for. It's to build what you need next.

Once the core system is stable (Phase 2), I propose three innovations to drive retention and automate the "boring stuff."

1. The "Nightly AI Auditor"

Problem: Humans miss unsigned documents.
Solution: An OCR script runs every night at 3 AM. It scans every new PDF for signatures. If a signature is missing, it auto-flags the learner and emails the Compliance Manager before the real auditor arrives.

Risk Reduction
2. WhatsApp "Nudge" Engine

Problem: Learners ignore compliance emails.
Solution: We integrate the Twilio API. If attendance drops below 85%, the system sends an automated WhatsApp: "Hi Alex, we missed you today. Is everything okay?"

Retention Boost
3. Auto-CV Matching

Problem: Finding jobs for learners is manual work.
Solution: The system parses the learner's CV skills (Python, SQL) and auto-matches them against live job APIs (Indeed/LinkedIn), presenting the Learner with a personalized "Jobs Feed" in their dashboard.

Workflow Architecture

Comparing Standard Requirements vs Proposed Innovations
Stage 1
Onboarding
Standard Requirement

1. Applicant submits data via Website.
2. Eligibility is checked manually or via basic rules.
3. Documents uploaded to portal.

My Innovation
The "AI Pre-Check"
  • OCR Scanner: AI reads the passport expiry date instantly. Rejects invalid docs before a human sees them.
  • Lead Triage: Ineligible leads are auto-routed to the Sales Team for commercial upselling.
Stage 2
Delivery
Standard Requirement

1. Tutors mark attendance registers.
2. Admin chases absent students via email.
3. Assignments submitted to LMS.

My Innovation
WhatsApp Nudge Engine
  • Auto-Chase: If marked 'Absent', system waits 1 hour then sends a WhatsApp message automatically.
  • Intervention Lock: If 2 assignments are missed, the system locks access to the next module until a Tutor call is booked.
Stage 3
Assessment
Standard Requirement

1. Tutor marks work.
2. Student views grade.
3. Grade is stored in database.

My Innovation
The "Dispute Ticket" System
  • Review Window: Grades are "Provisional" for 48 hours. Students must click "Accept" or "Raise Ticket".
  • Change Audit: If a tutor changes a grade later, they must select a 'Reason Code' which is logged for the government auditor.
Stage 4
Exit
Standard Requirement

1. Course completion check.
2. PDF Certificate generated.
3. File archived.

My Innovation
AI Career Booster
  • Smart Certificate: PDF includes a QR code. Employers scan it to verify authenticity against the live database (Anti-Fraud).
  • AI CV Updater: System appends verified skills to the student's CV automatically.
